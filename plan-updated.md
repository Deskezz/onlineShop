# План разработки: Демо-сайт онлайн-магазина электроники (v3.0 — AI-Optimized)

**Цель:** Создать полнофункциональный демо-магазин электроники для портфолио. Минималистичный дизайн, строгая архитектура, клиентская работа с данными, мультиязычность (RU/EN), тёмная/светлая тема. Код должен демонстрировать уровень Middle/Senior: типобезопасность, производительность, UX-детали, отсутствие архитектурного дрейфа.

---

## 1. Стек технологий
| Категория | Технология | Примечание |
|---|---|---|
| Фреймворк | Next.js 14+ (App Router) | SSR/SSG, `server-only` / `client-only` разделение |
| Язык | TypeScript (`strict: true`) | Без `any`, без `@ts-ignore` |
| Стилизация | Tailwind CSS 3.4+ + `class-variance-authority` + `tailwind-merge` | Замена тернарников в `className` |
| Состояние | Zustand + `persist` | Только для клиентских стейтов (корзина, избранное, тема, язык) |
| Анимации | CSS transitions + Framer Motion (только микро-анимации) | **Без page transitions** (сложность в App Router) |
| Иконки | Lucide React | SVG, tree-shakeable |
| Шрифты | `Inter` + `Inter Tight` (`next/font`) | 0 FOUT/FOIT |
| Изображения | `next/image` + локальные файлы | `public/assets/products/`, предсказуемые пути |
| i18n | `next-intl` | App Router, `[locale]` routing, `getTranslations` / `useTranslations` |
| Формы | React Hook Form + Zod | Типобезопасная валидация |
| Данные | Статические TS-файлы (`data/`) | Нормализованная структура, разбиты по сущностям |
| Линтинг | ESLint + Prettier | Чистый код, единый стиль |

## 2. Концепция дизайна
**Философия:** Минимализм (Apple / B&O / Aesop). Много воздуха, крупная типографика, акцентные детали. Каждый пиксель оправдан.
**Палитра (Soft Obsidian):**
- Light: Фон `#F8F9FA`/`#FFFFFF`, Текст `#1A1C1E`/`#64748B`, Акцент `#0F172A`, Границы `#EDF2F7`
- Dark: Фон `#0F1115`/`#1C1F26`, Текст `#F1F5F9`/`#94A3B8`, Акцент `#F1F5F9`, Границы `#2D333D`
**Типографика:** `Inter Tight` (заголовки, 600-700, `tracking-tight`), `Inter` (body, 400, `leading-relaxed`).
**Компоненты:** Радиус `16px` (карточки), `8px` (кнопки), `9999px` (бейджи). Тени только в light теме. В dark — `1px` border. Переходы `300ms cubic-bezier(0.4,0,0.2,1)`.
**Антипаттерны:** Никакого `#000`, градиентов, теней в dark, анимаций >300ms, `lorem ipsum`, UI-библиотек (shadcn/mui), тернарников в `className`.

## 4. Страницы и функционал (ключевые требования)
**Главная:** Hero (статичное фото, без карусели), CategoriesGrid, FeaturedProducts, PromoSection.
**Каталог:** Sidebar (категория, цена, бренд, рейтинг) + Grid (3/2/1). Сортировка, пагинация, бейджи активных фильтров. Мобильные фильтры → drawer + `useScrollLock`.
**Карточка товара:** Галерея, инфо (варианты: цвет/память), +/-, добавление в корзину/избранное, спецификации, отзывы, похожие товары.
**Корзина:** Список, изменение количества, удаление, промокод `DEMO` (-10%), sticky summary, переход к checkout.
**Checkout:** 3 шага (доставка → оплата → подтверждение). Zod-валидация. Предзаполнение демо-данными. Пометка "Это демо".
**Успешный заказ:** Анимация, номер, кнопки навигации. Заказ сохраняется в `ordersStore`.
**Профиль:** Демо-аккаунт автоавторизован. Обзор, список заказов, детали заказа с таймлайном, избранное.
**Поиск:** Модалка (Cmd+K), debounce 300ms, до 6 результатов, `normalizeQuery()`, закрытие по Esc/клику вне.
**SEO (базовое):** `<title>`, `<meta description>`, Open Graph для ключевых страниц. Реализуется на этапе полировки.

## 5. 🔒 КРИТИЧЕСКИЕ ПРАВИЛА ДЛЯ ИИ (ПРИОРИТЕТ ВЫШЕ ВСЕГО)
1. **Серверные/Клиентские компоненты:** `'use client'` только в листьях (интерактив, формы, модалки, сторы). Страницы, Layout, сетки товаров, галереи — серверные. Запрещено импортировать стейт-сторы в серверные компоненты.
2. **Гидратация:** Все `persist`-сторы читаются **исключительно** через `useHydratedStore<T>()`. До монтирования клиента компонент возвращает `Skeleton` или `null`. Прямой вызов стора в рендере запрещён.
3. **Нормализация состояния:** Корзина и избранное хранят **ТОЛЬКО** `{ productId, variantKey?, quantity }`. Запрещено сохранять `name`, `price`, `image`, `locale`. При рендерите данные резолвятся из `data/` по `id` с учётом `locale`.
4. **Стили:** Все варианты компонентов (`Button`, `Badge`, `Input`, `Card`) пишутся через `cva`. Классы объединяются **только** через `cn()`. Тернарные операторы в `className` запрещены.
5. **Данные:** Моки разбиты на 3 файла. `ordersStore` при инициализации мержит статические заказы из `data/orders.ts` с сохранёнными в `localStorage`.
6. **Поиск:** Использует `normalizeQuery()` (`toLowerCase().trim().normalize('NFD').replace(/[\u0300-\u036f]/g, '')`). Транслитерация не требуется, поиск регистронезависимый.
7. **Скролл:** Все оверлеи (поиск, фильтры, мобильное меню) обязаны использовать `useScrollLock()` при открытии.
8. **Изображения:** Все фото локальные. Пути в `products.ts`: `['/assets/products/iphone-15-pro-1.webp']`. Использовать `next/image` с явными размерами.
9. **Git / GitHub — АБСОЛЮТНЫЙ ЗАПРЕТ ДЛЯ ИИ.** Репозиторий создан и связан с GitHub пользователем вручную. ИИ **КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО** выполнять любые git-команды: `git add`, `git commit`, `git push`, `git checkout`, `git merge` и т.д. Все коммиты и пуши делает **ТОЛЬКО** пользователь вручную после тестирования каждого этапа.

## 6. Этапы разработки (Чекпоинты)
**ЭТАП 1:** Фундамент. Next.js, TS, Tailwind, `cva`+`twMerge`, `next-intl`, тема, структура, `useHydratedStore`, `cn()`, базовый SEO.
**ЭТАП 2:** Дизайн-система. UI-компоненты через `cva`, скелетоны, тосты, рейтинг.
**ЭТАП 3:** Layout. Header (sticky/hide), Footer, MobileMenu, Breadcrumbs, Demo-banner.
**ЭТАП 4:** Данные и стейт. Нормализованные моки (`products`, `specs`, `reviews`, `orders`). Stores с `persist`. Логика мержа заказов. Демо-пользователь.
**ЭТАП 5:** Главная страница. Hero, Categories, Featured, Promo. ProductCard.
**ЭТАП 6:** Каталог. Grid, FilterSidebar, ActiveFilters, Sort, Pagination. Фильтры на мобильном (drawer + scroll lock). Клиентская фильтрация/сортировка.
**ЭТАП 7:** Карточка товара. Gallery, Info, Specs, Reviews, Related. Выбор варианта, +/-, add to cart/fav.
**ЭТАП 8:** Корзина. Items, Summary, Promo (`DEMO`), Empty state. Sticky summary.
**ЭТАП 9:** Checkout. 3 шага, Zod, предзаполнение, пометка "демо". `checkout-success` с анимацией. Очистка корзины.
**ЭТАП 10:** Профиль. Sidebar, Overview, Orders list, Order details (timeline), Favorites.
**ЭТАП 11:** Поиск. Modal, Cmd+K, debounce, результаты, пустые состояния, scroll lock.
**ЭТАП 12:** Полировка. Hover/focus, микро-анимации (CSS/Framer), a11y, 404, favicon, базовые meta/OG теги, `npm run build` проверка, финальный `progress.md`.

## 7. Инструкции для ИИ-разработчика
1. **Поэтапность.** Делай строго по этапам. После завершения — остановись, обнови `progress.md`, жди подтверждения.
2. **Остановка при проблемах.** Если ошибка не решается за 2 попытки, есть неоднозначность или риск отклонения от архитектуры → **СТОП**. Спроси пользователя. Не додумывай.
3. **Формат вывода.**
   - Выводи **ПОЛНОЕ** содержимое каждого нового/изменённого файла. Никаких `// ... остальной код`.
   - Никаких `// TODO`, `// FIXME`, закомментированного кода.
   - Файл компонента ≤ 150 строк. Если больше — разбей на подкомпоненты.
   - При создании компонента сразу добавляй экспорт в barrel-файл (если используется).
4. **Контекст.** Перед началом этапа читай `progress.md` и этот план. Не переписывай работающее. Правь точечно.
5. **Запреты.** `any`, `as any`, UI-библиотеки, прямой `localStorage` вне `persist`, локальные строки в коде (всё через `next-intl` или `data/`), внешние ссылки на изображения, page transitions через Framer Motion.

## 8. Критические требования к качеству
- Строгий TypeScript. Все типы определены.
- Чистые импорты через `@/`.
- Каждое действие имеет обратную связь (toast, UI-состояние).
- Пустые состояния оформлены.
- Все иконки-кнопки имеют `aria-label`, поддерживают `focus-visible` и навигацию клавишами.
- Базовые мета-теги (`<title>`, `description`, `og:image`) заполнены.
- `npm run build` проходит без ошибок и варнингов.
